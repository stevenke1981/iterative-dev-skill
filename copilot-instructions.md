# Copilot 迭代開發規則（Iterative Development Rules）

## 核心鐵律

1. **預設模式** — 本專案處於「持續迭代開發」狀態，禁止整份重寫既有檔案。
2. **差異更新** — 只輸出「新增」「修改」「刪除」的部分，標註版本號（如 `v1.3 新增`）或使用 diff 格式。
3. **強制讀取** — 修改前嘗試讀取：`plan.md`, `spec.md`, `blueprint.md`, `knowledge_graph.md`, `readme.md`, `ITERATIVE_DEV_CORE.md`
4. **版本追蹤** — 每次修改後，在相關檔案新增/更新 `## 變更歷史` 區塊（日期、內容、影響範圍）。
5. **錯誤自動處理** — 用戶貼上錯誤訊息（error/exception/stack trace/FAILED）時，自動定位根因並輸出最小 patch，不重寫整個檔案。

## 回應開頭格式（強制）

每次回應開頭必須輸出：
```
【迭代開發模式已啟用】
核心文件讀取狀態：[plan.md:已讀取/不存在] [spec.md:...] [blueprint.md:...] ...
本次更新：[檔案名] → [新增/修改/刪除]
```

## diff 輸出格式

```diff
## [檔案名] — v[版本號] 差異更新

+ 新增的內容（標註版本）
- 刪除的舊內容
  不變的內容
```

或使用標註格式：
```markdown
<!-- v1.x 新增 START -->
新內容
<!-- v1.x 新增 END -->
```

## 知識圖譜更新

每次新增功能時，輸出 `knowledge_graph.md` 的差異更新：
```markdown
節點：[新功能名稱]
關係：[新功能] → depends_on → [既有模組]
```

## 停用條件

只有用戶明確說「從零開始重寫」或「切換成全新開發模式」時，才暫停迭代開發規則。
