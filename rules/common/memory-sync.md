# MEMORY_SYNC — 自動記憶更新規則（v1.0）

> 本文件為 iterative-dev.md 的子模組，專注於自動記憶同步。

## 觸發時機

每次迭代完成後（Step 6），自動執行以下記憶寫入。

## 記憶寫入規則

### project 記憶（每次迭代必寫）

```markdown
---
name: iterative-[專案名]-v[版本號]
description: [專案名] 第 [版本號] 次迭代摘要
type: project
---

本次迭代版本：[版本號]
日期：[YYYY-MM-DD]
影響檔案：[file1, file2, ...]
迭代摘要：[一句話描述]

**Why:** [為何需要這次迭代]
**How to apply:** [下次迭代的注意事項]
```

### feedback 記憶（錯誤場景才寫）

當本次迭代是因為錯誤訊息觸發時：

```markdown
---
name: error-fix-[錯誤類型簡稱]
description: [錯誤類型] 的解法與防範策略
type: feedback
---

錯誤類型：[Error 類型或描述]
根因：[根本原因]
解法：[採用的修復方式]

**Why:** 用戶貼上此錯誤，迭代開發技能自動介入修復。
**How to apply:** [下次遇到類似錯誤的處理方式]
```

## 記憶更新鐵律

1. 不重複寫入相同版本的記憶（先檢查是否已存在）
2. 若已存在同版本記憶，執行差異更新（不覆蓋）
3. 記憶描述必須夠具體，供未來對話直接使用

## 記憶索引維護

每次新增記憶後，同步更新 `~/.claude/projects/.../MEMORY.md` 索引，
格式：`- [記憶名稱](file.md) — 一行摘要（版本號、日期）`
