# VERSION_TRACKING — 版本追蹤規範（v1.0）

> 本文件為 iterative-dev.md 的子模組，專注於版本號管理。

## 版本號格式

```
v{major}.{minor}.{patch}
```

| 層級 | 遞增時機 |
|------|---------|
| major | 重大架構變更、不相容的 API 變更 |
| minor | 新增功能（向後相容） |
| patch | Bug 修復、小幅修改 |

## 變更歷史區塊格式（強制）

每個被修改的檔案，必須在文件末尾維護此區塊：

```markdown
## 變更歷史
| 版本 | 日期 | 內容 | 影響範圍 |
|------|------|------|----------|
| v1.2 | 2026-04-15 | 新增 X 功能 | module_a, module_b |
| v1.1 | 2026-04-14 | 修復 Y Bug | module_a |
| v1.0 | 2026-04-13 | 初始建立 | — |
```

## diff 標註格式

輸出差異時，必須標註版本來源：

```diff
+ 這是 v1.2 新增的內容
- 這是 v1.1 已廢棄的內容（v1.2 移除）
  這是不變的內容
```

或使用 HTML 注釋標記：
```markdown
<!-- v1.2 新增 START -->
新內容
<!-- v1.2 新增 END -->

<!-- v1.2 修改 START（原 v1.1 內容：舊描述） -->
修改後內容
<!-- v1.2 修改 END -->
```

## 版本追蹤鐵律

1. 絕不刪除歷史版本記錄（只新增）
2. 每次迭代必須遞增版本號（至少 patch）
3. `ITERATIVE_DEV_CORE.md` 的 `current_version` 必須同步更新
